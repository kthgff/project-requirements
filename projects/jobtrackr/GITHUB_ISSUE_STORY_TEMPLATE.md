# JobTrakr GitHub Issue Story Template

Use this template for every active GitHub issue.

## User Story
As a <user / engineer / system>, I want <capability> so that <outcome>.

## Context
- Why this matters now
- What part of the product or delivery flow this affects
- Any relevant upstream docs or contracts

## Detail
- Clear description of the intended change
- Scope boundaries
- Key behaviors or implementation expectations
- Important constraints

## Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Definition of Done
- [ ] Code or doc changes completed
- [ ] Relevant tests or validation completed
- [ ] Integrated with affected flow
- [ ] Ready for QA or review

## Notes
- Legacy ticket ID
- Owner
- Branch
- Dependencies / blockers
- Links to source-of-truth docs
